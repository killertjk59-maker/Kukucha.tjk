# Kukucha.tjk — интернет-магазини кӯдакона (Flask)

Лоиҳаи одӣ ва воқеан коркунанда бо Flask + PostgreSQL, тайёр барои Railway.

## Функсияҳо

- Витринаи маҳсулот: саҳифаи асосӣ, категорияҳо, саҳифаи маҳсулот, ҷустуҷӯ
- Сабат, дӯстдоштаҳо, профил, таърихи фармоишҳо
- Фармоиш → пардохт бо кортии бонкӣ (боргузории скриншоти чек) → тасдиқи админ
- Панели админ: маҳсулот, категория, фармоиш (тасдиқ/рад/тағйири статус), мизоҷон, статистика
- Bottom navigation (5 тугма), дизайни кремӣ/беж

## Насб дар Railway (қадам ба қадам)

1. **Репозиторийи GitHub созед** ва ҳамаи ин файлҳоро дар он бор кунед (ё бевосита аз Railway "Deploy from GitHub repo"-ро истифода баред).
2. Дар [railway.app](https://railway.app) лоиҳаи нав созед → **New Project → Deploy from GitHub repo**.
3. Дар ҳамон лоиҳа **+ New → Database → Add PostgreSQL** илова кунед. Railway худ тағйирёбандаи `DATABASE_URL`-ро ба сервиси веб пайваст мекунад (агар пайваст накунад, аз "Variables" онро дастӣ илова кунед).
4. Ба сервиси веб равед → **Variables** → инҳоро илова кунед:
   - `SECRET_KEY` — як сатри тасодуфии дароз
   - `ADMIN_PHONE` — рақами телефони ҳисоби админ (масалан 992000000000)
   - `ADMIN_PASSWORD` — рамзи муваққатии админ
   - (ихтиёрӣ) `ADMIN_CARD_NUMBER`, `ADMIN_CARD_HOLDER`, `ADMIN_CARD_BANK`
5. Railway худ `requirements.txt`-ро мебинад ва `Procfile`-ро барои оғози сервер истифода мебарад (`gunicorn app:app`).
6. Пас аз аввалин деплой, дар Railway → сервиси веб → **Settings → тугмаи "..." → Open Shell** (ё аз бахши "Deploy Logs" истифода баред) як маротиба фармони зеринро иҷро кунед, то ҳисоби админ ва категорияҳо сохта шаванд:

   ```
   python seed.py
   ```

   Ин ҳисоби админро бо `ADMIN_PHONE`/`ADMIN_PASSWORD`-и шумо месозад ва 5 категорияи аслӣ (Либос, Пойафзол, Бозича, Навзод, Аксессуар) илова мекунад.

7. Ба сомонаи худ ворид шавед, бо рақами телефон ва рамзи админ дар қадами 4 **Ворид шудед** → аз "Профил" ба **Панели админ** гузаред → маҳсулот илова кунед.

## Санҷиши маҳаллӣ (агар компютер дошта бошед)

```bash
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env       # ва СEC RET_KEY-ро иваз кунед
python seed.py
python app.py
```

Сайт дар `http://localhost:5000` кушода мешавад. Бе `DATABASE_URL`, лоиҳа худкор ба SQLite мегузарад (файли `kukucha.db`), пас барои санҷиши локалӣ Postgres лозим нест.

## Дар бораи расмҳо (муҳим)

Ин версия расмҳоро дар папкаи `static/uploads/` дар худи сервер нигоҳ медорад. Railway-и стандартӣ **файлсистемаи доимӣ надорад** — ҳангоми ҳар деплойи нав, расмҳои боргузошташуда гум мешаванд.

- Барои сомонаи воқеан production, тавсия дода мешавад минбаъд ба **Railway Volume** (диски доимӣ) ё ба хидматҳои беруна (Cloudinary, S3, ва ғ.) гузаред.
- Барои оғози кор ва санҷиш, ин система комилан кофист.

## Сохтори лоиҳа

```
app.py              # app factory, оғози сервер
config.py           # танзимот (DATABASE_URL, корти пардохт)
extensions.py       # db, login_manager, csrf
models.py           # User, Product, Category, Order, ...
routes/
  main.py           # витрина, ҷустуҷӯ
  auth.py           # вуруд/сабти ном
  cart.py           # сабад, дӯстдоштаҳо
  orders.py         # фармоиш, пардохт, профил
  admin.py          # панели админ
templates/          # HTML (Jinja2 + Tailwind CDN)
static/              # CSS ва расмҳои боргузошташуда
seed.py              # сохтани ҳисоби админ ва категорияҳои аслӣ
```

## Огоҳиномаҳо (Realtime Notifications)

Версияи ҳозира огоҳиномаро тавассути flash-паёмҳо дар браузер нишон медиҳад (масалан пас аз тасдиқи фармоиш). Огоҳиномаи воқеии real-time (push/SMS/Telegram bot) дар версияи ҳозира нест — метавон онро дар марҳилаи оянда илова кард (масалан бо Telegram Bot API барои огоҳ кардани админ).
